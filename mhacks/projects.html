Meanie Chrome meaniechrome In a CCDC round, I was interested when there was apparently a malicious Chrome extension in someone's follow-up talk (one of the red teamers) so I figured I'd make a basic script to inject a js file into a remote browser to see what conventional security is there to stop it. (HSTS helps if attacker is using HTTP, access policy helps but chrome extensions can make themselves excepted, etc). At the end of the day it was extremely successful and could probably be used in a training image. Requires developer mode. Link
Cyber Scripts cyber-scripts In collaboration with Justice Suh, some different scripts where development meets exploitation. Link
PHP Web Shell php-web-shell Justice Suh started a repo (or maybe I did and he helped) where we could deliver a basic PHP file as a remote shell (like c99) and I thought I could make some improvements. Keeping it lean and covert, but still usable. It's supposed to look like a very generic login form, but the username is converted to base64, then the PHP script decodes it and executes it in shell. Very basic but fun for CTFs. Intentionally basic so it's as convert as possible and I can write it down and bring it into competitions. Link
CyberPatriot Scorer scoring An unofficial Linux practice scoring engine for CyberPatriot. Intentionally insecure so image makers can go back and debug, also it's practice so if you cheat in practice that's your fault. I made the bash implementation, then the other gcma-cp contributers insisted it'd be easier to write in Python (despite the fact that I wrote the bash implementation pretty easily, their choice though) so hence the blank python folder. Link
Academy Portfolio academyportfolio.github.io Marshall Academy wanted their ambassadors to make a portfolio to present to professionals in related industries. I figured I'd rather self-host than use Google Sites, especially if I'm presenting myself as a front-end developer Public Link Project Link
rumpwn rumpwn Rumkin is a popular online service used for basic CTF problems. It's a front-end Javascript implementation of a bunch of ciphers. I took a few (i.e. Caesar) and automated it so if I run caesar_pwn("contenttodecode") in the developer console, it'd run all 26 possibilities. I decided to expand it to stuff like automatic file carving and ultimately made a separate prettier platform called "CTF Toolkit" since nothing that process-intensive should occur on the client side. I don't think I ever open sourced it however. Link
